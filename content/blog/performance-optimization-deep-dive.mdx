## Optimization Strategy

### Phase 1: CSS Optimization (65% Reduction)

**Problem:**
- Mixed design systems (Discord + custom)
- Duplicate styles across components
- No systematic token usage
- Dead CSS from removed components

**Solution:**

1. **Token-based system:**
```css
/* Before: Scattered values */
.button { background: #5865F2; }
.card { border: 2px solid #2f3136; }

/* After: Tokenized */
:root {
  --nb-surface: #f2f1eb;
  --nb-border: #1a1918;
  --nb-accent: #5865F2;
  --nb-shadow-color: rgba(0, 0, 0, 0.15);
}

.component {
  background: var(--nb-surface);
  border: 2px solid var(--nb-border);
}
```

2. **Component cleanup:**
   - Removed 85+ unused CSS classes
   - Consolidated duplicate button/card/input styles
   - Shared utility classes via `@apply` in Tailwind
   - Eliminated inline styles

3. **Result:**
   - CSS: 2800 lines → 800 lines (65% reduction)
   - Initial CSS payload: -40KB

### Phase 2: Code Splitting & Lazy Loading

**Problem:**
- All components loaded upfront
- Heavy animations in main bundle
- Music player blocking initial render
- Lottie animations parsed synchronously

**Solution:**

1. **Lazy component loading:**
```typescript
// Before
import { MusicPlayer } from '@/components/blog/MusicPlayer'
import FooterLottie from '@/components/home/FooterLottie'

// After
import dynamic from 'next/dynamic'

const LazyMusicPlayer = dynamic(
  () => import('@/components/LazyMusicPlayer'),
  { 
    ssr: false,
    loading: () => <div className="h-20" />
  }
)

const LazyFooterLottie = dynamic(
  () => import('@/components/home/LazyFooterLottie'),
  { ssr: false }
)
```

2. **Component file splitting:**
```typescript
// Before: All in one file
// app/page.tsx (1400+ lines)

// After: Split by concern
// components/home/HomeHeader.tsx
// components/home/HeroContent.tsx
// components/home/HeroBackdrop.tsx
// components/home/HomeFooter.tsx
// components/home/neo/NeoProjectsSection.tsx
// components/home/neo/NeoFeatureGrid.tsx
```

3. **Music player optimization:**
   - Moved to separate lazy-loaded component
   - Audio context initialization deferred
   - State persisted to localStorage (prevents re-initialization)

4. **Results:**
   - Initial JS bundle: -80KB
   - Time to Interactive: -1.2s
   - First Load JS: 180KB → 140KB (main page)

### Phase 3: Component Architecture

**Problem:**
- Monolithic component files
- Mixed concerns (layout + logic + data fetching)
- No clear separation of primitives

**Solution:**

1. **Primitive components:**
```typescript

  return <button className={neoButtonVariants({ variant, size })} {...props} />
```

2. **Layout components:**
```typescript

// components/home/HeroContent.tsx (content + layout)
// components/home/HomeHeader.tsx (navigation)
```

3. **Data separation:**
```typescript

// lib/data/neo-home.ts (content config)
// components/*/ (presentation only)
```

**Results:**
- Better tree-shaking
- Faster hot reloads in development
- 60% smaller individual component files

### Theme Persistence

```typescript
// components/theme-provider.tsx
export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState<'light' | 'dark'>('light')
  
  useEffect(() => {
    // Load from localStorage (prevents flash)
    const saved = localStorage.getItem('theme')
    if (saved) setTheme(saved)
  }, [])
  
  useEffect(() => {
    // Persist preference
    localStorage.setItem('theme', theme)
    document.documentElement.setAttribute('data-theme', theme)
  }, [theme])
  
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}
```

## Performance Best Practices Applied

### 1. Critical Path Optimization

```typescript
// app/layout.tsx
export default function RootLayout({ children }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <head>
        {/* Preload critical fonts */}
        <link
          rel="preload"
          href="/fonts/inter.woff2"
          as="font"
          type="font/woff2"
          crossOrigin="anonymous"
        />
      </head>
      <body>
        <ThemeProvider>
          {children}
        </ThemeProvider>
      </body>
    </html>
  )
}
```

### 2. Lazy Hydration

```typescript
// Defer non-critical hydration
import { useEffect, useState } from 'react'

function useHydration() {
  const [hydrated, setHydrated] = useState(false)
  
  useEffect(() => {
    setHydrated(true)
  }, [])
  
  return hydrated
}

// Usage
function HeavyComponent() {
  const hydrated = useHydration()
  
  if (!hydrated) return <Skeleton />
  
  return <ActualComponent />
}
```

### 3. State Persistence

```typescript
// Prevent expensive re-initialization
function MusicPlayer() {
  const [volume, setVolume] = useState(() => {
    if (typeof window === 'undefined') return 0.7
    const saved = localStorage.getItem('music-volume')
    return saved ? parseFloat(saved) : 0.7
  })
  
  useEffect(() => {
    localStorage.setItem('music-volume', volume.toString())
  }, [volume])
  
  // Volume restored instantly, no re-calculation
}
```

### 4. Intersection Observer for Lazy Loading

```typescript
function useInView(threshold = 0.1) {
  const [inView, setInView] = useState(false)
  const ref = useRef<HTMLDivElement>(null)
  
  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setInView(true)
          observer.disconnect()
        }
      },
      { threshold }
    )
    
    if (ref.current) observer.observe(ref.current)
    
    return () => observer.disconnect()
  }, [threshold])
  
  return [ref, inView]
}

// Usage
function LazySection() {
  const [ref, inView] = useInView()
  
  return (
    <div ref={ref}>
      {inView ? <HeavyContent /> : <Placeholder />}
    </div>
  )
}
```

## Lessons Learned

### What Worked Best

1. **Lazy loading**: Biggest impact on initial load time
2. **CSS tokenization**: Reduced bundle + improved maintainability
3. **Component splitting**: Better tree-shaking and DX
4. **Mobile-first refactor**: Critical for Core Web Vitals

### What Didn't Work

1. **Premature optimization**: Started with heavy refactoring before profiling
2. **Manual audits**: Tooling catches things humans miss
3. **One-time optimization**: Performance degrades over time, needs monitoring

### Best Practices Going Forward

1. **Performance budget**: Set and enforce max bundle size
2. **Continuous monitoring**: Track Core Web Vitals in production
3. **Lazy by default**: Load only what's needed, when needed
4. **Measure first**: Profile before optimizing
